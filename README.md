# Discord Bridge Bot 🤖🌐 (SQLite Cloud Edition)

Универсальный Discord-бот для связывания текстовых каналов между независимыми серверами. Поддерживает два режима работы: классическую однонаправленную трансляцию (например, логов правок с MediaWiki) и создание сквозных межсерверных кросс-чатов для объединения вики-союзов.

Эта ветка использует реляционную облачную базу данных **SQLite Cloud (sqlite.ai)** вместо Redis, сохраняя высокую скорость работы, полную структуру связей таблиц и удобную веб-панель управления. Для отправки сообщений применяются **Discord Webhooks**, что позволяет полностью копировать никнеймы и аватарки исходных авторов.

---

## 🚀 Основные возможности

* **Режим single:** Классический мост в одну сторону (Канал А -> Канал Б). Идеально для трансляции системных логов, обновлений или правок от других ботов. Привязка целей осуществляется напрямую по ID канала-источника.
* **Режим cross:** Полноценная сеть межсерверных чатов (Канал А <-> Канал Б <-> Канал В). Сообщения синхронизируются между всеми участниками сети "веером" через текстовое имя комнаты.
* **Маскировка вебхуков:** Бот автоматически создаёт вебхуки на целевых серверах, копируя аватарки, никнеймы пользователей и поддерживая вложения (картинки/файлы).
* **Защита от бесконечного эха (Looping):** Встроенные проверки предотвращают зацикливание пересылки сообщений между вебхуками.

## 📊 Структура базы данных (SQLite Cloud)

В отличие от NoSQL-ключей, в этой ветке данные разложены по классической реляционной схеме из двух таблиц с поддержкой каскадного удаления:

1. **`bridges`** — Хранит метаданные созданных мостов.
   * `bridge_id` (TEXT, PRIMARY KEY) — ID канала-источника (для single) или уникальное текстовое имя (для cross).
   * `mode` (TEXT) — Тип моста (`single` или `cross`).
2. **`bridge_channels`** — Связывает каналы с мостами.
   * `id` (INTEGER, AUTOINCREMENT) — Первичный ключ записи.
   * `bridge_id` (TEXT, FOREIGN KEY) — Ссылка на родительский мост (с `ON DELETE CASCADE`).
   * `channel_id` (TEXT) — ID подключенного Discord-канала.

## 🛠️ Архитектура и зачем нужен Веб-сервер (aiohttp)

В коде бота параллельно с Discord-клиентом запускается легковесный асинхронный веб-сервер на **aiohttp**, слушающий порт **10000**. 

### Зачем это нужно?
Этот сервер выполняет критически важную задачу для бесплатного хостинга (например, **Render**):

1. **Предотвращение "засыпания" (Keep-Alive):** Бесплатные веб-сервисы автоматически засыпают, если к ним не поступает входящих HTTP-запросов в течение 15 минут. Если бот уснёт, он перестанет реагировать на сообщения.
2. **Прохождение пингов (Health Check):** Хостинг периодически стучится по HTTP на указанный порт, чтобы проверить, жив ли процесс. Благодаря aiohttp, бот успешно отвечает кодом 200 OK, подтверждая стабильность системы.

## 📋 Команды бота

Управление мостами происходит через современные слэш-команды Discord (требуются права **Администратора**):

* **/bcreate mode:single source:[ID канала]** — Инициализировать мост трансляции правок для указанного канала-источника.
* **/bcreate mode:cross name:[Имя моста]** — Инициализировать глобальную сеть кросс-чатов с уникальным текстовым именем.
* **/bconnect bname:[Имя моста или ID источника] channel:[ID канала, опционально]** — Подключить канал к системе. Если аргумент `channel` не указан, к мосту привязывается текущий текстовый канал, в котором была вызвана команда.
* **/bdelete name:[Имя моста или ID источника]** — Полностью удалить мост/источник из базы данных (каскадно очистив все связанные с ним каналы) и деактивировать пересылку.

## ⚙️ Установка и запуск

### 1. Переменные окружения
Создайте в корне проекта файл **variables.env** (он автоматически скрыт через .gitignore) и заполните его следующими строками:

* `DISCORD_TOKEN` = ваш_токен_дискорд_бота
* `SQLITE_CLOUD_CONNECTION_STRING` = строка_подключения_вида_sqlitecloud://admin:pass@host.sqlite.cloud:8860/database

### 2. Установка зависимостей
Выполните команду в терминале:
```bash
pip install -r requirements.txt
```

### 3. Запуск бота
Выполните команду в терминале:
```bash
python bot.py
```
