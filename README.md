# Discord Bridge Bot 🤖🌐

Универсальный Discord-бот для связывания текстовых каналов между независимыми серверами. Поддерживает два режима работы: классическую однонаправленную трансляцию (например, логов правок с MediaWiki) и создание сквозных межсерверных кросс-чатов для объединения вики-союзов.

Проект использует **Upstash Redis** для мгновенного обмена ID каналов и **Discord Webhooks** для сохранения никнеймов и аватарок исходных авторов сообщений.

---

## 🚀 Основные возможности

* **Режим single:** Классический мост в одну сторону (Канал А -> Канал Б). Идеально для трансляции системных логов, обновлений или правок от других ботов.
* **Режим cross:** Полноценная сеть межсерверных чатов (Канал А <-> Канал Б <-> Канал В). Сообщения синхронизируются между всеми участниками сети "веером", создавая единое пространство для общения.
* **Полная совместимость:** Новая архитектура кросс-чатов полностью изолирована и не ломает уже настроенные ранее мосты.
* **Маскировка вебхуков:** Бот автоматически создаёт вебхуки на целевых серверах, копируя аватарки, никнеймы пользователей и поддерживая вложения (картинки/файлы).
* **Защита от бесконечного эха (Looping):** Встроенные проверки предотвращают зацикливание пересылки сообщений между вебхуками.

## 🛠️ Архитектура и зачем нужен Веб-сервер (aiohttp)

В коде бота параллельно с Discord-клиентом запускается легковесный асинхронный веб-сервер на **aiohttp**, слушающий порт **10000**. 

### Зачем это нужно?
Этот сервер выполняет критически важную задачу для бесплатного хостинга (например, **Render**):

1. **Предотвращение "засыпания" (Keep-Alive):** Бесплатные веб-сервисы на Render автоматически засыпают, если к ним не поступает входящих HTTP-запросов в течение 15 минут. Если бот уснёт, он перестанет реагировать на сообщения в Дискорде.
2. **Прохождение пингов (Health Check):** Хостинг периодически стучится по HTTP на указанный порт, чтобы проверить, жив ли процесс. Благодаря aiohttp, бот успешно отвечает кодом 200 OK, подтверждая, что система работает стабильно, и хостинг не перезапускает контейнер принудительно.

## 📋 Команды бота

Управление мостами происходит через современные слэш-команды Discord (требуются права **Администратора**):

* **/bcreate mode:single source:[ID канала] target:[ID канала]** — Создать классический однонаправленный мост из одного канала в другой.
* **/bcreate mode:cross name:[Имя моста]** — Инициализировать новую пустую комнату (сеть) для глобального кросс-чата.
* **/bconnect bname:[Имя моста] channel:[ID канала]** — Подключить текущий канал к ранее созданному кросс-мосту.

## ⚙️ Установка и запуск

### 1. Переменные окружения
Создайте в корне проекта файл **variables.env** (он автоматически скрыт через .gitignore) и заполните его следующими строками:

* DISCORD_TOKEN = ваш_токен_дискорд_бота
* UPSTASH_REDIS_REST_URL = ссылка_на_ваш_upstash_redis
* UPSTASH_REDIS_REST_TOKEN = токен_вашего_upstash_redis

### 2. Установка зависимостей
Выполните команду в терминале: **pip install -r requirements.txt**

### 3. Запуск бота
Выполните команду в терминале: **python bot.py**
