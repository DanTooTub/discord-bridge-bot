# Discord Channel Bridge Bot 🚀

Простой и эффективный Discord-бот на Python, предназначенный для автоматической пересылки сообщений из одного канала в другой в реальном времени. 

> 🤖 **Важное примечание:** Этот проект полностью спроектирован, написан и оптимизирован с помощью искусственного интеллекта (**Gemini**) по ТЗ разработчика.

---

## 🔥 Ключевые функции

* **Single-server пересылка:** Моментальная трансляция сообщений между текстовыми каналами внутри одного Discord-сервера.
* **Кросс-серверная пересылка:** Возможность пересылать сообщения между абсолютно разными серверами (главное, чтобы бот присутствовал на обоих).
* **Умная работа с контентом:**
  * Обычный текст пересылается чистым текстом с красивым указанием автора сообщения и сервера-источника (`**Ник [Сервер]:** текст`).
  * Прикрепленные файлы, картинки и документы пересылаются вслед за текстом.
* **Пересылка встроенных карточек (Embeds):** Если исходное сообщение отправлено другим ботом или вебхуком и содержит Embed, наш бот подхватит его структуру и перешлет в целевой канал один в один в чистом виде.

## 🛠️ Стек технологий и особенности архитектуры

* **Язык:** Python 3.12+ (протестировано на Python 3.13)
* **Библиотеки:** `discord.py`, `python-dotenv`
* **Безопасность:** Токен бота и ID каналов не хардкодятся в файлах конфигурации, а безопасно считываются из переменных окружения (`variables.env`).
* **24/7 Бесплатный хостинг:** В код встроен легковесный асинхронный веб-сервер на базе `http.server`. При развертывании на бесплатных облачных платформах (например, Render) это позволяет использовать внешние пингаторы (типа UptimeRobot) для имитации трафика, что предотвращает «засыпание» сервера на бесплатных тарифах.

## ⚠️ Известные ограничения и проблемы (Rate Limits)

При активном использовании бота вы можете столкнуться со следующей особенностью API Discord:

* **Задержка при флуде (HTTP 429):** У Discord установлены жесткие лимиты на частоту отправки сообщений в один канал (5 сообщений в 5 секунд). При интенсивном спаме или массовой пересылке сложных сообщений (включая Embed-карточки и тяжелые медиафайлы) в консоли бота будут появляться предупреждения `We are being rate limited`.
* **Как это обрабатывается:** Архитектура библиотеки `discord.py` автоматически перехватывает эти ограничения. Бот **не падает и не теряет сообщения**, а выстраивает их в очередь и отправляет сразу же, как только спадет ограничение от Discord. Из-за этого при флуде сообщения могут приходить в целевой канал с небольшой задержкой (в несколько долей секунды/секунд).

## ⚙️ Требования для работы

Для корректной работы бота на стороне Discord Developer Portal должны быть включены:
1. **Интенты:** Вкладка *Bot* -> Раздел *Privileged Gateway Intents* -> Должен быть включен тумблер **Message Content Intent**.
2. **Права бота:** В целевых каналах боту необходимы права на просмотр канала, отправку сообщений, встраивание ссылок (`Embed Links`) и прикрепление файлов.
